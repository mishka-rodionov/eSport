# Модуль Feature: Center

Модуль `center` является центральным функциональным блоком приложения, отвечающим за организацию и управление соревнованиями по спортивному ориентированию. Он объединяет в себе инструменты для организаторов, позволяя сопровождать мероприятие на всех этапах: от создания до подведения итогов.

## Основной функционал

### 1. Создание и редактирование соревнований
*   **Конструктор соревнований:** Интерфейс для настройки базовых параметров (название, дата, время, место проведения, описание).
*   **Управление группами:** Добавление и настройка возрастных и спортивных групп (М21, Ж18 и т.д.), установка параметров дистанции (длина, количество КП, контрольное время).
*   **Выбор системы отметки:** Поддержка различных систем (например, Sportiduino).

### 2. Управление участниками
*   **Регистрация:** Возможность ручного добавления участников в соответствующие группы.
*   **Редактирование данных:** Оперативное изменение информации об участниках (ФИО, группа) через удобный интерфейс вкладок.
*   **Список участников:** Отображение всех зарегистрированных атлетов с сортировкой по стартовым номерам.

### 3. Жеребьевка
*   **Автоматизированный процесс:** Случайное распределение стартовых номеров и времени старта.
*   **Умное перемешивание:** Алгоритм жеребьевки минимизирует вероятность старта участников из одной группы друг за другом, обеспечивая справедливость соревнований.

### 4. Работа с оборудованием (NFC/Чипы)
*   **Выдача чипов:** Интерфейс для привязки номеров электронных чипов к участникам перед стартом.
*   **Чтение карт:** Функционал для считывания данных с NFC-карт участников.
*   **Контроль прохождения:** Инструменты для отметки на контрольных пунктах и проверки статуса участника.

### 5. Результаты и аналитика
*   **Просмотр результатов:** Отображение текущих протоколов по группам.
*   **Учет времени:** Отображение общего времени прохождения дистанции, ранга и статуса (FIN, DNF и т.д.).

## Технологический стек
*   **UI:** Jetpack Compose (Material 3).
*   **Архитектура:** MVI-подобный подход (State, Action, ViewModel) с использованием Base-классов проекта.
*   **DI:** Koin.
*   **Navigation:** Модульная навигация на базе Type-Safe Compose Navigation.

## Структура пакетов
*   `data/`: Модели состояния (`State`), действий (`Action`) и интеракторы бизнес-логики.
*   `presentation/`: Composable-экраны и соответствующие ViewModels для каждого подраздела (создание, список участников, жеребьевка, выдача чипов и т.д.).
